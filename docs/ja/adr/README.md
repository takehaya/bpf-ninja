# ADR（設計判断の記録）

このディレクトリは、xdp-ninja のアーキテクチャ上の判断を記録する。
採用した決定と、検討したうえで却下した選択肢の両方を、同じ台帳に残す。

採用した決定を残すのは、後から「なぜこの構造にしたか」を辿れるようにするためである。
却下した選択肢を残すのは、それがコードにもコミット履歴にも残らないからである。
記録しなければ、同じ道を後から誰か（将来の自分を含む）がもう一度たどり、同じ測定をして同じ結論に達する。
この再探索を防ぐ。

## 何を書くか

各記録は冒頭に `状態:` を持つ。値は次のいずれか。

- **採用**：この決定にしたがって実装されている。
- **却下**：検討したが採用しなかった。
- **置換**：かつて採用したが、別の決定で置き換えた（置換先の番号を書く）。

本文には、検討した変更とその動機、実際に測った数値と条件、判断とその根拠、トレードオフと後から効きそうな別の方向を書く。
憶測ではなく計測で判断する。

連番は追加順で、欠番があってもよい。
